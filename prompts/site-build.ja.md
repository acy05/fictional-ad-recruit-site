# Codex Prompt: Site Build

次の条件で、架空の広告会社の採用サイトを実装してください。

## Goal

- 静的サイトとして完成させる。
- 既存構成は `index.html`、`styles.css`、`script.js`、`assets/generated/`。
- 会社名、人物、案件、数字、コピーはすべて架空化する。
- 外部サイトの画像、SVG、動画、CSS、JSは使わない。

## Design Direction

- 白背景。
- 黒アウトライン。
- コーラル、シアン、アンバー、ミントをアクセントに使う。
- 大きな英字見出しと日本語コピーを組み合わせる。
- 手描き風イラスト断片を画面全体に配置する。
- 採用サイトらしい明るさは出すが、説明過多なランディングページにしない。

## Sections

1. FV
   - `MADO CREATIVE RECRUIT` 相当の大きなタイトル。
   - 複数のイラスト断片を周囲に配置。
   - 初回表示で中央起点から断片が展開し、少し遅れてタイトルが描画される。

2. Manifesto
   - 左に大きい日本語コピー。
   - 右に企画プロセスの円形サムネイル6点。
   - 背景に薄い街や広告のイラスト。

3. PEOPLE
   - 中央見出し。
   - 5人ループの人物スライダー。
   - 波線、円、六角形のフレーム。
   - 黒丸の前後ボタン。

4. PROJECT
   - 赤系の大きな背景エリア。
   - 架空案件カード4枚。
   - hoverで軽く持ち上がる。

5. NUMBERS
   - 左に見出しと説明。
   - 右に3枚の大きな数字カード。
   - スクロールで重なりカードが展開する。

6. ENTRY
   - 新卒と中途のCTA。
   - PCは3Dフリップ風hover。
   - モバイルはカードが下から浮き上がる表示。

## Constraints

- 画像は `prompts/image-assets.ja.md` に沿って新規生成する。
- ロゴ、本文、数値、ナビは画像化しない。
- `prefers-reduced-motion: reduce` に対応する。
- PCとモバイルの両方で横スクロールを出さない。
- 実装後は `docs/qa-checklist.md` を使って確認する。
